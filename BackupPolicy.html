Backup Policy


The policy is part of the Business Resiliency Program and supports the company’s Disaster Recovery Plan (DRP).


1. Backup Policy Overview


Scope: All SM2ARTA production data (PostgreSQL databases, Vault encryption keys, and configuration files) hosted on the OVH VPS environment.


Frequency: Full daily encrypted backups are automatically executed at midnight (Frankfurt time).


Location: Backups are stored on a dedicated OVH backup node physically separate from the production VPS.


Encryption: All backup archives are encrypted using AES-256 and protected by access-controlled credentials stored in Vault OSS.


Retention: Backups are retained for 30 days on a rolling basis.


Access Control: Only the System Administrator (Managing Member) has authorization to initiate restores or access backup storage.


2. Backup Completion Verification (Requirement a)


The backup process generates an automatic completion log and checksum report after every run.


Logs are reviewed weekly to confirm successful execution and integrity validation.


Any failure automatically triggers an email alert to the System Administrator.


A monthly summary of successful backups is recorded in the Security & Maintenance Log.


3. Backup Testing and Restoration Verification (Requirement b)


Test restorations are performed on the staging environment quarterly to verify backup integrity and measure restoration time.


During each test, databases and configurations are restored to a sandbox container to confirm completeness and consistency.


The most recent test (recorded in the DR log) confirmed that full restoration could be achieved in under 12 hours (RTO) with a maximum of 24 hours’ data loss (RPO).


Results of each test are documented in the Backup Verification Log.


4. Governance and Review


Policy Owner: System Administrator, SM2ARTA LLC.


Review Frequency: Annual review or upon major infrastructure change.


Backup automation scripts and logs are version-controlled in GitHub and reviewed quarterly for updates.
