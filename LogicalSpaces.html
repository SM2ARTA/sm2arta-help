Logical Spaces

Overview


Logical Spaces represent abstract locations based on the function or purpose of the space rather than its physical location.


At early planning stages, exact locations may not be known — however, the functional requirement for each space is already defined and must be fulfilled to host the event.


These requirements often need to be collected early to ensure timely sourcing, production, and delivery of FF&E and related materials.


The Category and Subcategory are mandatory fields when creating a Logical Space — you must have both created in the system before adding any new Logical Spaces.


Logical Spaces Categories and Subcategories


Logical spaces follow a fixed three-layer structure in SM2ARTA:


  • Logical Space Category
    • Logical Space Subcategory
      • Logical Space


Access permissions can be assigned at different levels — Logical Space, Subcategory, or Category — depending on user responsibilities. 


Access granted at the Category level automatically includes access to all related Subcategories and Logical Spaces, and therefore prevails over lower-level permissions.


Logical Space Code


Each logical space has a unique Logical Space Code that can be referenced across multiple departments such as Overlay, IT, Signage, and others.


This standardization ensures that all teams can identify and coordinate around the same space without confusion.


Logical Space Code can either be automatically generated by SM2ARTA or manually entered by the user if a predefined coding structure is required.


The automatically generate Space code option creates codes in the format CategoryCode.SubcategoryCode.SpaceCode.The Space Code must be unique across the entire directory.
