Change Management


Change Management in SM2ARTA provides a controlled mechanism for managing updates to finalized Material Requests.


It allows Functional Material Planners to submit Change Requests for revised Requested quantities, while Material Planning Managers maintain full control over reviewing, approving, or declining these changes.


This process ensures that all modifications after the Block Planning phase are tracked, auditable, and justified, preventing inconsistencies between planning and delivery stages.


Activation


Blocking an Event does not automatically enable Change Management. The feature must be manually activated to give Material Planning Managers flexibility — particularly for historical or completed events, where planning may remain blocked but further change control is unnecessary.


Once Change Management is enabled:


  • A new column called Change request appears in the Material Planning, allowing Functional Material Planners to submit updated Requested quantities;
  • The ability to add/remove new Material Requests to already existing Logical Spaces is also restored;
  • Any new quantities entered are stored in the Change Request column, while the original “Requested” quantity remains unchanged for reference until Change Request is approved.


To support the process, the following columns are added to the Material Plan when Change Management is active to project the impact of changes and track the requests statuses:


Column

Description

Change request

The new Requested quantity submitted by the Functional Material Planner.

Projected delta

The difference between the originally Requested quantity and the Change Request quantity.


Positive means that Requested quantity will be reduced.

Negative means that Requested quantity will be increased.

Status

Indicates the current state of each Change Request.


Change Request Statuses


To manage the lifecycle of each request, SM2ARTA uses a defined set of statuses:


Status

Description

Created

Default status of all Material Requests in the Material Plan before any changes are submitted.

Pending

The Change Request has been submitted by a Functional Material Planner.

Declined

The Change Request was reviewed and rejected by the Material Planning Manager.

Confirmed

The Change Request was confirmed and approved by the Material Planning Manager.

Partially Confirmed

Only part of the requested change was approved by the Material Planning Manager.


These statuses help maintain full transparency throughout the approval process.


Change Request Principles


The Change Management process in SM2ARTA is based on the principle of redistributing materials within the same Venue or Venue Cluster to maintain balance and operational efficiency without unnecessary re-sourcing.


A Venue Cluster represents a geographically grouped set of venues, typically connected through the same distribution center or logistics network. It is assumed, that within a cluster, assets can be easily redistributed between venues until they are dispatched for delivery.


There are two primary types of Change Requests:


  • Increase – when a Functional Material Planner requests a higher quantity of a specific material than initially planned;
  • Decrease – when a Functional Material Planner reduces the quantity previously requested, generating a positive delta that can potentially be used to fulfill increases from other projects within the same venue or cluster.


Example Scenario:


  • Project A decreases its requirement for plastic folding chairs from 100 to 50 units, creating a positive delta of 50;
  • Project B, in the same Venue or Cluster, submits a Change Request for 50 additional chairs;
  • The positive delta from Project A offsets Project B’s increase, enabling the Material Planning Manager to redistribute materials efficiently without additional procurement.


Material Planning Manager Control


The Material Planning Manager has full oversight of the process and can:


  • Review each submitted Change Request in the Material Plan;
  • Approve, decline, or partially approve individual requests;
  • Use the Change Management Report to track all Accumulated deltas and Pending change requests in one consolidated view;
  • Perform bulk approvals through the Change Management Report.


Change Management Report


While Material Plan provides information on current Change Requests, it does not give you information on the Accumulated deltas (All Materials available for redistribution). For this purposes, when Change Management is enabled, SM2ARTA activates a dedicated Change Management Report.


This report provides Material Planning Managers with a centralized dashboard to:


  • View positive Accumulated materials quantity across Venues and Clusters;
  • View all consolidated Pending change requests;
    • Positive value means the decreased Change Request - Materials will be released ;
    • Negative values means the increased Change Request - Materials will be redistributed.
  • Identify where positive accumulated deltas can offset increases;
  • Execute bulk approvals when sufficient deltas are available to cover multiple requests.

The Change Request approval process defines how Material Requests are updated based on the Material Planning Manager’s decision.


Each outcome — Declined, Approved, or Partially Approved — updates the corresponding Material Request line according to specific system rules to ensure accurate data tracking and auditability.

If Change Request is Declined


When a Change Request (line 001) is Declined by the Material Planning Manager:


  • The Status of the Materials Request line changes to Declined;
  • The Change request Quantity is reset to blank;
  • The Requested quantity remains unchanged (no update is applied);
  • The Projected delta is reset to blank, as the new Requested quantity now matches the approved value;
  • The Material Request line is removed from the Change Management Report if there are no remaining Change Requests associated with that Material.

If Change Request is Partially Approved


Partial approval applies only to Change Requests that propose an increase in quantity.


When a Change Request is partially approved:


  • The Status changes to Partially confirmed;
  • The Change request quantity remains the same, as only a portion of the request was approved;
  • The Requested quantity increases by the number of partially approved materials taken from the Accumulated delta;
  • The Projected delta quantity remains as the difference between the new Requested quantity and the Change request quantity;
  • The Accumulated materials decreases by the number of approved materials. In this case, it should be reduced to 0, as all available materials were used to partially fulfill the request.

Disabling Change Management


Disabling Change Management does not delete or alter any existing data. It simply hides the Change Management interfaces (columns and reports) while preserving all underlying information.


All submitted and processed Change Requests remain in the system for reference and audit purposes.
