About the System

System Overview


SM2ARTA (System for Materials Management and Asset Tracking) is a cloud-based platform developed to support material planning, asset tracking, and logistics coordination for large-scale events.


The system allows users to:


  • Consolidate material requirements across multiple venues and functional areas
  • Plan, monitor, and optimize procurement, storage, and dissolution workflows
  • Track materials and assets throughout their lifecycle
  • Generate real-time dashboards and operational reports for decision-making
  • Easily manage continuous changes in material requirements, ensuring that all updates are instantly reflected across relevant planning, reporting, and logistics modules


System Architecture and Technology Stack


SM2ARTA uses modern, open-source technologies designed for high performance, reliability, and security:


  • Back-end Framework: PHP 8.1 (Laravel 10)
  • Database Engine: PostgreSQL (12.1-alpine)
  • Operating System: Linux (Ubuntu)
  • Containerization: Docker-based micro-service infrastructure
  • Web Server: NGINX within Docker containers
  • Email Relay: SendGrid — used exclusively for sending password reset links
  • Authentication: Laravel Sanctum (API sessions), Spatie Permission (RBAC), Lab404 Impersonate (admin login as user)


Each service runs within its own Docker container, allowing isolation, scalability, and simplified updates during development and production deployment.


Hosting and Infrastructure


SM2ARTA is hosted on OVH Cloud Virtual Private Servers (VPS) located in the OVH Frankfurt (Germany) data center.


The hosting environment offers:


  • ISO/IEC 27001–certified and GDPR-compliant infrastructure
  • Encrypted SSD storage with automated backups
  • OVH Edge Network Firewall for inbound traffic filtering
  • Continuous uptime and performance monitoring
  • Secure access through encrypted SSH and VPN connections


The German hosting location ensures data residency within the European Union, compliant with EU data protection regulations.


Web Application Firewall (WAF)


All inbound traffic to SM2ARTA is routed through the Cloudflare Web Application Firewall (WAF), which acts as a secure global proxy between users and the application.


Cloudflare provides:


  • Protection against web vulnerabilities (SQLi, XSS, CSRF, etc.)
  • DDoS and bot mitigation
  • IP masking of backend servers (origin IPs remain hidden)
  • Enforced HTTPS and TLS 1.3 encryption


Only verified and filtered traffic from Cloudflare’s global edge network can reach the SM2ARTA servers, ensuring security and high availability.


User Access and Authentication


SM2ARTA uses a role-based access control (RBAC) model to manage user privileges effectively:


  • Material Planner: System administrator with full access.
  • Functional Material Planner: Functional area representative with permissions to manage materials within their scope.
  • Venue Logistics Manager: Advanced user with limited access to Material Planning and Asset Tracking modules.


Additional safeguards include:


  • Two-Factor Authentication (2FA)
  • Encrypted credentials and secure password storage
  • Pre-activation workflow (users are configured before activation)


Data Security and Protection


All personal and operational data are encrypted in transit (via HTTPS/TLS 1.3) and at rest (AES-256).


SM2ARTA complies with GDPR data protection principles and enforces strict access control policies.


Personal data (such as names, emails, and contact numbers) is pseudonymized or encrypted using secure key vaults to protect user privacy.


Data Residency and Compliance


All system data is stored exclusively in the OVH Frankfurt (Germany) data center, ensuring compliance with European Union (EU) data protection laws.


Cross-border access from the United States and other regions occurs securely through encrypted HTTPS sessions routed via Cloudflare’s edge network.


SM2ARTA adheres to international standards of confidentiality, integrity, and lawful processing of data in line with GDPR requirements.

For data protection purposes, any Export to Excel operation that includes personal data is automatically logged.


The Administrator must provide a reason before initiating such an export to ensure traceability and compliance with data privacy requirements.


Data Integrity and System Protection


SM2ARTA is designed with strict data integrity safeguards to protect the consistency and accuracy of operational information.


Certain data entities—such as Events, Users, Projects, Venues, and Materials—cannot be removed once they are linked to other records or transactions.


This restriction prevents the accidental deletion of dependent data, such as planning entries, material allocations, or asset tracking transactions.


When removal could compromise data relationships, the corresponding Delete icons are automatically disabled, ensuring that all historical information and audit trails remain intact.


These measures guarantee that every change made in SM2ARTA is traceable, preserving the reliability of planning, tracking, and reporting data across all tournaments and operational phases.


Change Management and Deployment Control


All configuration and code changes are tracked via Git and follow documented change management procedures:


  • Code updates undergo peer review and approval before deployment.
  • Automated CI/CD pipelines handle testing and controlled releases.
  • Deployment logs and version tags ensure complete release traceability.
  • Rollback procedures are in place to quickly revert to prior versions if needed.


This ensures that only approved and verified changes are applied to production environments.


Logging, Monitoring, and Incident Response


All system and user activities — including logins, exports, and administrative actions — are logged with user IDs and timestamps.


  • Logs are securely stored and retained per compliance requirements.
  • Access to logs is restricted to authorized administrators.
  • Alerts are generated for suspicious access patterns or failed login attempts.


Each incident is documented, tracked, and analyzed to prevent recurrence.


Development and Test Environment Segregation


SM2ARTA maintains separate development, staging, and production environments.



  • Each environment runs in isolated Docker containers with its own database.
  • Only anonymized or synthetic data is used in test environments.
  • Deployment to production requires explicit administrative approval.


This segregation prevents unauthorized data exposure and ensures stable deployments.


System Maintenance and Monitoring


The SM2ARTA technical team continuously monitors system health, performance, and security.


Routine maintenance includes:


  • Regular OS and software updates
  • Docker container patch management
  • Scheduled vulnerability scans
  • Encrypted off-site backups
  • Comprehensive audit trails and performance monitoring


In addition, automated vulnerability scans are periodically performed to identify potential security risks in the application.


These authorized tests are conducted using approved third-party tools such as HostedScan.io and OWASP ZAP, under controlled conditions and with restricted test accounts.


All vulnerability testing is carried out in a dedicated test environment, separate from production servers, to ensure that live operations are never affected.


Alignment with SOC 2 Trust Services Criteria


The controls described above align with the AICPA Trust Services Criteria:


  • Security: Protection of systems against unauthorized access or disclosure
  • Availability: Continuous operation and monitored performance
  • Confidentiality: Secure handling of sensitive and proprietary data
  • Processing Integrity: Accurate and complete data processing
  • Privacy: Responsible handling of personal data in line with GDPR principles
