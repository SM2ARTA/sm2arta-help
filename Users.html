Users

Overview


Users represent the individual accounts that access SM2ARTA.


Each user is assigned specific roles, projects, and access rights that determine what they can view or edit within the system.


Assigning users correctly ensures accurate permissions, secure collaboration, and clear accountability across all functions.


The total number of users is displayed on the Main Dashboard (managed through Editting User Details).


User Roles


SM2ARTA includes several user roles that define different levels of access and responsibility within the system:


  • Material Planner


System administrator with full access to all SM2ARTA functions.


Can manage users, create and edit directories, assign privileges, and oversee all events, venues, and material plans.


  • Functional Material Planner


Representative of a Functional Area (Project) responsible for adding and managing materials within their assigned scope.


Access is limited to events, venues, and catalogs based on assigned privileges.


  • Venue Logistics Manager


Advanced user with extended but limited access to the Material Planning and Asset Tracking sections.


Can view, update, and monitor materials in line with venue-related access privileges.


Each user must be assigned the appropriate role according to their operational responsibilities and required visibility.


User Creation and Activation


Users are created in the system but remain inactive by default.


This approach allows administrators enough time to configure and verify the user’s access privileges before granting system access.


Once all privileges are set, the user must be activated to gain entry into SM2ARTA.


Two-Factor Authentication (2FA)


Two-Factor Authentication (2FA) adds an extra layer of security for user accounts.


2FA can be enabled or disabled for each user through the Editing User Details.


When 2FA is enabled, the user will be required to verify their identity using a code sent to his registered email.
